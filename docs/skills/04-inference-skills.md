# Skill: Inference Pipeline (Phase 4)

This document provides guidelines for VRAM optimization and pipeline orchestration during runtime.

## 1. VAE Tiling
- **Why**: Large images or dense reconstruction during VAE decoding can trigger memory spikes that exceed 2 GB VRAM.
- **How**: Enable VAE tiling to decode the image in smaller latent patches:
  ```python
  pipeline.enable_vae_tiling()
  ```

## 2. Pipeline Caching
- **Avoid Reloading**: Loading models from disk takes significant time. Store the loaded pipeline in a singleton class or global cache.
- **Unloading**: When switching functions (e.g., executing Python-level image manipulation or running U²-Net), unload the pipeline:
  ```python
  del pipeline
  gc.collect()
  torch.cuda.empty_cache()
  ```

## 3. NSFW Toggle implementation
- **Logic**: The application supports an optional clothes removal function.
- **Mechanism**:
  - When **NSFW Toggle is FALSE**: Load `lora_fashion.safetensors`, use standard fashion clothing prompts, and target the clothing mask generated by U²-Net.
  - When **NSFW Toggle is TRUE**: Load `lora_nsfw.safetensors`, change target prompt to body-reconstruction tokens (e.g., skin details, anatomy realism), and adjust target mask to expand slightly beyond clothes boundaries for realistic blending.
- **Base safety**: Disable the default `StableDiffusionSafetyChecker` only in private offline environments if it conflicts with NSFW recreation.
